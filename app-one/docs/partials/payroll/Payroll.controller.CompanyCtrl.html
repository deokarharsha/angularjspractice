CompanyCtrl
controller in module Payroll

Description

A company controller is used to store and retrieve company details. A company controller is attached to the 'company.html' page via the ng-controller directive, AngularJS will instantiate a new Controller object, using the specified Controller's constructor function. A new child scope will be created and made available as an injectable parameter to the companyCtrl constructor function as $scope.

The fields of company page are stored under $scope.data json object. Due to which we can send collection of json to post service. Here in company page we had performed add & edit operations using same view template. Means if $scope.data.ID = 0 then its an add mode and if it is not then it will be work in edit mode.

Methods

  • SelectData()

    Description:This method is used to select cities from the select list. This is an auto complete for get cities where we have used 'angucomplete-alt' custom directive. This method contain sub-method for selectedCity.

    Method Parameters: tablename(i.e. Cities) and columnname(i.e. name).

    Used Stored Procedure: getSelectData

  • SelectData()

    Description: This is again the same method which we have discussed above but the difference is that it is an auotcomplete list for states. which contains sub-method for selectedState.

    Method Parameters: tablename(i.e. States) and columnname(i.e. name).

    Used Stored Procedure: getSelectData

  • edit()

    Description: This method retrieves the record of the company of specified ID for the edit purpose. After retrieving specific record it rebinds that details on the company form. This method also contains two broadcast methods which are used to rebind the angucomplete-alt's record like here cities and states.

    *Note: An angucomplete-alt has one loophole that it does not support two way data binding thats why we need to use broadcast method.

    Method Parameters: tablename(i.e. Companies) and ID of the record which we want to retrieve(i.e. id).

    Used Stored Procedure: getEditData

  • reset()

    Description: This method resets all form data. And broadcast method is used to clear the angucomplete-alt's data. After resettting data now ID will be 0 so now the company page will be in add mode.

  • submit()

    Description: This method updates company details into the companies table, which uses stored procedure 'doCompanies' with parameters in $scope.data. This method works in two ways like if the $scope.data.ID = 0 i.e. add mode then it saves data to the table but if $scope.data.ID != 0 then it updates the given record.

    Used Parameters: stored procedure name(i.e. doCompanies) and form data(i.e. $scope.data)
